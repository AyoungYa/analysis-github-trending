# Report Templates

默认使用中文输出。用户要求英文或中英双语时再切换。

## Daily Trending Brief

```markdown
# GitHub Trending 技术简报

时间范围: [daily / weekly / monthly]
筛选条件: [语言 / 主题 / 无]
数据可信度: [A/B/C/D，说明来源]

## 一句话结论

[用 2-3 句话说明今天最重要的趋势，不要罗列项目名。]

## 重点项目

| 项目 | 方向 | 评分 | 为什么火 | 建议 |
| --- | --- | ---: | --- | --- |
| [owner/repo](url) | [方向] | [0-100] | [核心原因] | [学习/试用/观察/跳过] |

## 趋势信号

- [信号 1: 技术方向 + 证据]
- [信号 2: 用户需求 + 证据]
- [信号 3: 生态变化 + 证据]

## 值得深入的项目

### 1. [owner/repo]

- 定位: [它解决什么问题]
- 亮点: [技术或产品亮点]
- 成熟度: [README / release / issue / commit / license]
- 风险: [主要不确定性]
- 适合谁: [开发者 / 创业者 / 研究者 / 团队]
- 下一步: [具体行动]

## 噪声和风险

- [哪些项目可能只是短期热点]
- [哪些结论数据不足]

## 行动建议

- 今天可以读源码: [repo]
- 可以试用集成: [repo]
- 加入观察列表: [repo]
- 暂时跳过: [repo]
```

## Repo Deep Dive

```markdown
# [owner/repo] 深度分析

仓库: [url]
分析日期: [date]
数据可信度: [A/B/C/D]

## 结论

评分: [0-100]
建议: [重点研究 / 试用 / 观察 / 暂不建议投入]

[用 3-5 句话给出核心判断。]

## 它解决的问题

[说明目标用户、使用场景和痛点。]

## 为什么进入 Trending

- [增长或传播原因]
- [技术/产品原因]
- [外部事件或生态原因]

## 技术判断

- 技术栈: [language/framework/runtime]
- 架构亮点: [如有]
- 可复用部分: [如有]
- 同类对比: [相对优势和短板]

## 工程成熟度

- 文档: [评价]
- 安装体验: [评价]
- 测试和 CI: [评价]
- release 和维护: [评价]
- issue / PR: [评价]
- license: [评价]

## 风险

- [风险 1]
- [风险 2]
- [风险 3]

## 适合人群

- 开发者: [是否值得学习]
- 团队: [是否值得引入]
- 创业者: [是否有产品机会]
- 研究者: [是否有研究价值]

## 下一步

1. [最小试用动作]
2. [需要验证的关键假设]
3. [跟踪指标]
```
